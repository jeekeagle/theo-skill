# 工作流步骤

本文档描述 theo-infocard 技能的工作流程。

## 流程图

```
[内容分析] → [密度评估] → [布局选择] → [代码生成] → [质量自检] → [完成]
```

## 详细步骤

### 步骤 1: 内容分析
- **输入**: 用户提供的内容描述或要点
- **处理**:
  - 识别核心主题和关键信息
  - 提取主要观点和次要信息
  - 确定内容类型（概念/数据/列表/混合）
- **输出**: 结构化的内容大纲

### 步骤 2: 密度评估
- **输入**: 结构化的内容大纲
- **处理**:
  - 计算信息点数量
  - 评估文本总量
  - 判断内容复杂度
  - 确定密度等级：低/中/高
- **输出**: 信息密度等级（附带 1 句话分析）

### 步骤 3: 布局选择
- **输入**: 信息密度等级
- **处理**:
  - **低密度** → 选择"大字符主义"布局
    - 超大标题（72-140px）
    - 充足留白
    - 极简装饰
  - **中密度** → 选择"平衡布局"
    - 标准标题层次
    - 清晰内容分区
    - 适度视觉元素
  - **高密度** → 选择"多栏网格"布局
    - 2-3 栏报纸式排版
    - 垂直分割线
    - 紧凑信息密度
- **输出**: 布局策略和结构方案

### 步骤 4: 代码生成
- **输入**: 布局策略和内容大纲
- **处理**:
  - 应用字体系统（Noto Serif SC / Noto Sans SC / Inter / Oswald）
  - 应用空间系统（外边距 40-50px，段落间距 ≤1.5em）
  - 应用视觉装饰（噪点纹理、重型分割线、背景色块）
  - 生成完整 HTML 结构
  - 生成内联 CSS 样式
  - 确保响应式设计
- **输出**: 完整的 HTML 文件（含内联 CSS）

### 步骤 5: 质量自检
- **输入**: 生成的 HTML 代码
- **处理**:
  - 检查正文字号（≥18px）
  - 验证行高（1.5-1.6）
  - 测试移动端可读性
  - 确认视觉层级清晰
  - 验证颜色对比度
- **输出**: 质量检查报告

### 步骤 6: 完成输出
- **输入**: 通过质量检查的 HTML 代码
- **处理**:
  - 格式化代码
  - 添加使用说明
  - 提供自定义建议
- **输出**: 最终的可交付成果

## 决策点

### 密度等级判断标准

| 密度等级 | 信息点数量 | 文本量 | 适用场景 |
|---------|----------|--------|---------|
| **低** | 1-2 个 | <100 字 | 单一概念、金句、口号 |
| **中** | 3-5 个 | 100-300 字 | 多个相关要点、小列表 |
| **高** | 6+ 个 | >300 字 | 详细说明、数据报告、长列表 |

### 布局选择依据

```
IF 密度 == "低" THEN
    布局 = "大字符主义"
    标题字号 = "120-140px"
    留白比例 = "60%+"

ELSE IF 密度 == "中" THEN
    布局 = "平衡布局"
    标题字号 = "56-72px"
    留白比例 = "40-50%"

ELSE IF 密度 == "高" THEN
    布局 = "多栏网格"
    标题字号 = "32-56px"
    留白比例 = "20-30%"
    栏数 = "2-3"
END IF
```

## 质量标准

### 必须满足
- ✅ 正文字号 ≥ 18px
- ✅ 行高 1.5-1.6
- ✅ 移动端可读性良好
- ✅ 颜色对比度符合 WCAG AA 标准
- ✅ 响应式设计

### 应该满足
- ✅ 视觉层级清晰
- ✅ 留白适当
- ✅ 字体一致性
- ✅ 代码整洁

### 可以满足
- ✅ 动画效果
- ✅ 交互元素
- ✅ 特殊视觉效果
