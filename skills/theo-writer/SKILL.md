# Theo Writer

智能写作协作工具 - 帮助你研究、大纲、起草和打磨内容，同时保持独特的声音和风格。

## 何时使用

- 撰写博客文章、通讯或简报
- 创建教育内容或教程
- 起草思想领导力文章
- 研究和编写案例研究
- 制作带引用的技术文档
- 撰写包含适当引用和参考文献的文章
- 改进开头和引言
- 写作时获得逐节反馈

## 核心功能

1. **协作大纲**：帮助将想法构建成连贯的大纲
2. **研究辅助**：查找相关信息并添加引用
3. **钩子改进**：强化开头以吸引注意力
4. **分段反馈**：在撰写时审查每个部分
5. **声音保持**：维护写作风格和语调
6. **引用管理**：正确添加和格式化参考文献
7. **迭代优化**：通过多稿改进

## 使用方法

### 基本工作流程

1. **从大纲开始**：
```
帮我创建关于 [主题] 的文章大纲
```

2. **研究并添加引用**：
```
研究 [具体主题] 并在我的大纲中添加引用
```

3. **改进钩子**：
```
这是我的引言。帮我让钩子更吸引人。
```

4. **获取分段反馈**：
```
我刚完成了"为什么重要"部分。审查并给出反馈。
```

5. **精炼和润色**：
```
审查完整草稿的流畅性、清晰度和一致性。
```

## 写作指导

当用户请求写作协助时：

### 1. 理解写作项目

询问明确问题：
- 主题和主要论点是什么？
- 目标受众是谁？
- 期望的长度/格式是什么？
- 目标是什么？（教育、说服、娱乐、解释）
- 有哪些现有研究或来源需要包含？
- 写作风格是什么？（正式、对话式、技术性）

### 2. 协作大纲

帮助构建内容结构：

```markdown
# 文章大纲：[标题]

## 钩子
- [开场白/故事/统计数据]
- [为什么读者应该关心]

## 引言
- 背景和语境
- 问题陈述
- 本文涵盖内容

## 主要部分

### 第一部分：[标题]
- 关键点 A
- 关键点 B
- 示例/证据
- [需要研究：具体主题]

### 第二部分：[标题]
- 关键点 C
- 关键点 D
- 需要数据/引用

### 第三部分：[标题]
- 关键点 E
- 反驳论点
- 解决方案

## 结论
- 主要观点总结
- 行动号召
- 最后的思考

## 研究待办
- [ ] 查找关于 [主题] 的数据
- [ ] 获取 [概念] 的示例
- [ ] 为 [声明] 来源引用
```

### 3. 进行研究

当用户请求对某个主题进行研究时：

- 搜索相关信息
- 查找可信来源
- 提取关键事实、引用和数据
- 按要求的格式添加引用

示例输出：
```markdown
## 研究：AI 对生产力的影响

关键发现：

1. **生产力提升**：研究表明内容创作任务节省 40% 的时间 [1]

2. **采用率**：67% 的知识工作者每周使用 AI 工具 [2]

3. **专家引用**："AI 增强而非取代人类创造力" - Dr. Jane Smith, MIT [3]

引用：
[1] McKinsey Global Institute. (2024). "The Economic Potential
    of Generative AI"
[2] Stack Overflow Developer Survey (2024)
[3] Smith, J. (2024). MIT Technology Review interview
```

### 4. 改进钩子

当用户分享引言时，分析和加强：

**当前钩子分析**：
- 有效的部分：[积极元素]
- 可以更强的部分：[改进领域]
- 情感影响：[当前 vs 潜在]

**建议的替代方案**：

选项 1：[大胆声明]
> [示例]
*为什么有效：[解释]*

选项 2：[个人故事]
> [示例]
*为什么有效：[解释]*

选项 3：[令人惊讶的数据]
> [示例]
*为什么有效：[解释]*

### 5. 提供逐节反馈

当用户撰写每一节时，审查：

```markdown
# 反馈：[章节名称]

## 做得好的地方 ✓
- [优势 1]
- [优势 2]
- [优势 3]

## 改进建议

### 清晰度
- [具体问题] → [建议修复]
- [复杂句子] → [更简单的替代方案]

### 流畅性
- [过渡问题] → [更好的连接]
- [段落顺序] → [建议重新排序]

### 证据
- [需要支持的声明] → [添加引用或示例]
- [通用声明] → [使其更具体]

### 风格
- [语调不一致] → [更好地匹配你的声音]
- [用词选择] → [更强的替代方案]
```

### 6. 保持作者的声音

重要原则：

- **学习他们的风格**：阅读现有写作样本
- **建议，不要替换**：提供选项，而不是指令
- **匹配语调**：正式、随意、技术性、友好
- **尊重选择**：如果他们更喜欢自己的版本，支持它
- **增强，不要覆盖**：让他们的写作更好，而不是不同

### 7. 引用管理

根据用户偏好处理参考文献：

**行内引用**：
```markdown
研究表明生产力提高 40% (McKinsey, 2024)。
```

**编号参考文献**：
```markdown
研究表明生产力提高 40% [1]。

[1] McKinsey Global Institute. (2024)...
```

### 8. 最终审查和润色

草稿完成后，提供综合反馈：

```markdown
# 完整草稿审查

## 总体评估

**优势**：
- [主要优势 1]
- [主要优势 2]
- [主要优势 3]

**影响**：[整体效果评估]

## 发布前检查清单
- [ ] 所有声明都有来源
- [ ] 引用格式化
- [ ] 示例清晰
- [ ] 过渡流畅
- [ ] 行动号召存在
- [ ] 校对错别字
```

## 专业提示

1. **在 VS Code 中工作**：比 web Claude 更适合长篇写作
2. **一次一节**：增量获得反馈
3. **单独保存研究**：保持 research.md 文件
4. **版本草稿**：article-v1.md, article-v2.md 等
5. **大声朗读**：使用反馈识别笨拙的句子
6. **设定截止日期**："我想今天完成草稿"
7. **休息**：写作，获得反馈，暂停，修订

## 文件组织

推荐写作项目结构：

```
~/writing/article-name/
├── outline.md          # 你的大纲
├── research.md         # 所有研究和引用
├── draft-v1.md         # 第一稿
├── draft-v2.md         # 修订稿
├── final.md            # 可发布版本
├── feedback.md         # 收集的反馈
└── sources/            # 参考材料
    ├── study1.pdf
    └── article2.md
```
